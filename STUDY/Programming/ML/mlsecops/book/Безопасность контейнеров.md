> [!security]
> Контейнеры также уязвимы, если их не защитить:  
> • Использовать доверенный контейнерный runtime (Docker, containerd)  
> • Обновлять runtime и образы  
> • Использовать user namespaces для изоляции  
> • Убирать лишние привилегии (`--cap-drop`)  
> • Использовать подписанный SBOM для анализа артефактов
